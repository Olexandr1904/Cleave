# Project Rename: Sickle/Tot → Cleave

**Date:** 2026-05-01  
**Status:** Approved

## Summary

Rename the project from "Sickle" (Python package / brand) and "tot" (repo/directory) to **"Cleave"** across the entire codebase, documentation, configuration, deployment artifacts, and GitHub.

**Rationale:** "Cleave" keeps the harvest/cutting metaphor (cuts through tickets, cuts developers out of routine work), sounds like a character name ("Cleave opened your PR"), is clean for open-source branding, and has no political or medical baggage. "Tot" was a meaningless working directory name.

## Execution Approach

Use `sed -i` for mechanical find/replace across file groups. Then handle special cases (file renames, git remote) manually.

Order:
1. Python package files (pyproject.toml, main.py)
2. Deploy artifacts (service file rename + content, deploy.sh)
3. Config defaults + examples
4. Agent files + claude commands
5. Root docs (README, CONTRIBUTING, CLAUDE.md)
6. All other docs (bulk sed)
7. Python source files (orchestrator, dashboard, integrations, config, workspace)
8. Test files
9. .claude/settings.local.json
10. Delete sickle.egg-info/, run `pip install -e .`
11. GitHub repo rename + git remote update

## Out of Scope

- Renaming the local directory `/home/admin0/tot` — this is a filesystem path. The user may want to do this manually. The Python package name and brand are what matter.
- Updating any live/deployed production environment — this spec covers the source repo only.
