# Requirements Validation Checklist

- [ ] Acceptance criteria exist in the ticket
- [ ] Each acceptance criterion is testable (has clear pass/fail condition)
- [ ] No vague language ("should work well", "be fast", "handle errors")
- [ ] All referenced entities/models are defined or exist in codebase
- [ ] Edge cases are identified or can be inferred
- [ ] Dependencies on other tickets are listed and their status is known
- [ ] Target files/modules can be identified from the requirements
- [ ] No conflicting requirements within the ticket
- [ ] Scope is bounded — clear what is NOT included
- [ ] Repo label is present for routing
